# めまい管理アプリ (memai_kanri)

日々の体調やめまい、その他の随伴症状を時間帯ごとにスコア化して記録し、自分自身の健康状態を可視化・管理するためのセルフケアWebアプリケーションです。

---

## 🌟 主な機能

*   **体調・症状のデイリー記録**
    *   各症状（めまい、ほてり、疲労感、睡眠状態など）について、時間帯（朝・昼・夜・全体）ごとに **なし(-1)**、または **0〜5段階** でスコアを入力できます。
    *   その日の特記事項や日記を記録できるフリーテキストメモ機能も備えています。
*   **症状のカスタマイズ（管理画面）**
    *   記録したい症状を自由に追加できます。
    *   各症状に対して「時間帯別の入力を有効にするか（有効にしない場合は『全体』のみの入力になります）」を設定できます。
    *   ドラッグ＆ドロップで記録画面に表示する症状の順番を直感的に変更可能です。
    *   不要になった症状は、過去の記録データを残したまま「非アクティブ（非表示）」に切り替えることができます。
*   **セキュアなユーザー認証**
    *   PBKDF2（SHA-256）を用いた安全なパスワードハッシュ化。
    *   `itsdangerous` による署名付きCookieセッションを用いた、セキュアなマルチユーザーログイン機能。

---

## 🛠️ 技術スタック

*   **バックエンド:** FastAPI (Python 3.9+)
*   **フロントエンド:** Vanilla HTML / CSS (CSSカスタムプロパティを用いたモダンデザイン) / JavaScript (非同期API通信)
*   **テンプレートエンジン:** Jinja2 (FastAPI統合)
*   **データベース:** SQLite3 (ファイルベースで環境構築が容易)
*   **ライブラリ:**
    *   `uvicorn`: 高速なASGIサーバー
    *   `itsdangerous`: 暗号化/署名付きセッションCookieの実現
    *   `python-multipart`: フォームデータの処理

## 📁 プロジェクト構成

```text
memai_kanri/
├── main.py                # アプリケーションのエントリーポイント、ルーティング、API定義
├── database.py            # SQLiteデータベースとの接続、テーブル初期化、各種CRUD処理
├── auth.py                # パスワード暗号化、セッション生成、認証用依存関係の定義
├── models.py              # アプリケーション全体の定数（時間帯の定義など）
├── create_user.py         # 新規ログインユーザーを追加するためのCLIスクリプト
├── requirements.txt       # 依存ライブラリ一覧
├── dev.md                 # 開発者用の環境構築および操作コマンドリファレンス
├── memai_kanri.service    # 本番デプロイ用のsystemdサービス定義
├── static/                # CSS、JavaScript、画像などの静的ファイル
└── templates/             # Jinja2によるHTMLテンプレートファイル
```

---

### 1. セットアップ
リポジトリをクローンまたは配置し、必要な依存関係をインストールします。

```powershell
# 依存関係のインストール
pip install -r requirements.txt
```

### 2. 管理者ユーザーの作成
アプリケーションを利用するためのユーザーを作成します。

```powershell
python create_user.py [希望するユーザー名] [希望するパスワード]
```

### 3. 開発サーバーの起動
Uvicornを使用してローカルサーバーを起動します。

```powershell
python -m uvicorn main:app --reload --port 8001
```

起動後、ブラウザで [http://localhost:8001](http://localhost:8001) にアクセスするとログイン画面が表示されます。作成したユーザー情報でログインしてください。

---

## 📋 データベースの管理

データは自動的に `data/health.db` (SQLite3) に保存されます。

*   **データのバックアップ:** `data/health.db` ファイルをコピーするだけで完了します。
*   **データベースの初期化:** `data/health.db` を削除した状態でサーバーを起動すると、テーブル構造およびデフォルトの症状（昨晩の睡眠、めまい、ほてり、疲労感、メンタル）が自動的に再生成されます。

---

## 🌐 本番環境へのデプロイ

Linuxサーバーなどにデプロイして常時稼働させる場合は、付属の `memai_kanri.service` を `/etc/systemd/system/memai_kanri.service` に配置し、systemdとして登録・管理することをお勧めします。

> [!NOTE]
> デプロイの際には、`auth.py` 内の `SECRET_KEY` を安全な固有の値に変更して運用してください。
